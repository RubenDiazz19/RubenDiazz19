## ⚡ Workflow Destacado: Jujutsu (JJ)

Uno de mis diferenciadores técnicos es el uso de **[Jujutsu (JJ)](https://github.com/martinvonz/jj)**, un sistema de control de versiones de nueva generación que redefine cómo se gestionan los estados de un repositorio.

```bash
# Ejemplo de workflow con JJ — sin staging area, sin caos
jj new          # nueva revisión anónima (como un commit, pero mejor)
jj describe     # añade descripción cuando el código está listo
jj rebase -d @- # reorganización limpia sin conflictos forzados
```

> 💡 **¿Por qué JJ?** Porque Git resuelve el problema de *guardar* cambios; JJ resuelve el problema de *pensar* en ellos. Las arquitecturas de estados que ofrece permiten flujos de trabajo mucho más limpios, especialmente en proyectos donde el historial importa tanto como el código.
